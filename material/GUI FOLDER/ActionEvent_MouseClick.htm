    Re: ActionEvent/MouseClick

    If you are going to use a MouseListener on a button then you probably should not use an ActionListener as well. The reason is some L&F implementations may fire an ActionEvent on mouseDown rather than on MouseUp (and some touch screen drivers implement mouse clicks on touch not release) in which case there is no guarantee the MouseListener would get the mouseDown event before the ActionEvent is fired.
    As for the addMouseListener(); what parameters should I call it with?
    An object that implements MouseListener. It's basically the same technique as when calling addActionListener().
    And after I called it; how can I get which mouse button was clicked on the JButton?
    The MouseEvent that is passed to the MouseListener methods has a getButton() method.

    Check out the API docs - all the classes are in the java.awt.event package.

    Re: ActionEvent/MouseClick

    You should pass a MouseListener to the addMouseListener() method. If you look up the MouseListener JavaDocs, you'll see it has quite a few mouse notification methods it can respond to. You only need the mouseClicked() method. To make life easier, a subclass of MouseListener is available, called MouseAdapter, that has 'do nothing' implementations of all those methods, so you can create a MouseAdapter and just override the method you want:
    Code:
    MouseListener myMouseListener = new MouseAdapter() {
   public void mouseClicked(MouseEvent event) {
      // your code here - event.getComponent() will give you the Component
      // that triggered the event.
   }
}

    Re: ActionEvent/MouseClick

    As you can see; I have called both actionListener and mouseListener for the button object.
    As I stated in my earlier post it's probably not safe to use both together - it may work on one machine but not on another.

    Dlorde explained which of the MouseListener methods to use ie MouseClicked(). You can keep using the MouseEntered() method as well if you need to do something when the mouse enters a component but to find out which button was clicked you need to use the MouseClicked() method.

    What you need to do is implement the mouseClicked() method and move all the code from the actionPerformed() method to the mouseClicked() method and then remove the ActionListener completely. The mouseClicked method can then call the mouseEvent's getButton() method to find out which button was clicked.
